Core Capabilities

Greenfield. Brownfield.
Everything in between.

Not a template generator. A project intelligence engine that works on any codebase.

AI-Powered Scaffolding

Describe your idea in plain English. Claude generates personas, features, architecture decisions, and a complete production-ready codebase.

Brownfield Analysis

vibe scan analyzes any existing project — detects stack, framework, missing best practices, and auto-generates a CLAUDE.md. No API key needed.

Modular Feature Injection

vibe add docker, vibe add ci, vibe add testing — inject production features into any project. Idempotent and stack-aware.

Project Health Scoring

vibe doctor scores your project with 17 weighted checks across 7 categories. Get a letter grade (A+ through F) with fix suggestions.

Context-Aware Coding

vibe run spawns Claude Code with your CLAUDE.md as context. Claude understands your architecture before writing a single line.

Read-Only Advisory

vibe ask lets you consult Claude about architecture, security, or performance — without making any file changes.

How It Works

Four steps. Full governance.

Prepare, build, anchor, audit — repeat.

vibe init

Generate 59 governance policies, CLAUDE.md instructions, auto-detected skills, and ADR templates. The foundation for governed AI coding.

vibe build

Describe your idea. Claude enriches it into personas, features, and architecture — then builds the project following all governance policies.

vibe anchor

Capture decisions in persistent context documents. Decisions table, constraints, open questions — survive across chat sessions.

vibe audit

10-category governance scorecard: security, accessibility, reliability, performance, compliance, clean code, API, data, code review, observability.

CLI Reference

Nine commands.
Full governance lifecycle.

Prepare, build, anchor, audit, analyze, enhance, code, and advise.

vibe init
--dry-run
Set up the vibe coding framework: CLAUDE.md, custom skills, guardrails, settings, and ADR template. Detects greenfield vs brownfield automatically.
vibe build
--verbose
Build a project from your idea. Enrichment generates personas, features, and architecture. Claude Code builds it using the CLAUDE.md framework.
vibe scan [dir]
--generate-claude-md
Analyze any existing project. Detects stack, framework, and best practice gaps. Auto-generates a CLAUDE.md. No API key needed for detection.
vibe add <feature>
--force --verbose
Inject production features: docker, ci, testing, logging, validation, health, hooks, auth, db. Claude-powered: api, component, model.
vibe doctor
--verbose
Governance compliance audit: 59 policies across 10 categories. Blocking violations, warnings, and quick-fix commands.
vibe anchor [feature]
--dry-run
Create or view feature context anchors. Persistent decision documents with decisions table, constraints, open questions, and state tracking.
vibe audit
--verbose
Alias for vibe doctor. Governance compliance audit with 59 policies, 10 categories, and severity levels (block/warn/info).
vibe run <task>
--verbose
Execute a coding task with Claude Code, enriched with your CLAUDE.md context. Ideal for features, refactoring, debugging, or any code change.
vibe ask <question>
--verbose
Ask Claude about your project in read-only advisory mode. No files are changed. Great for architecture decisions, security review, and trade-off analysis.

Quickstart

New Project (Greenfield)

$ npm install -g vibe-init-cli

# Prepare the room
$ mkdir my-app && cd my-app
$ vibe init

# Build from your idea
$ vibe build

# Check project health
$ vibe doctor
Existing Project (Brownfield)
# Set up framework for existing project
$ cd my-existing-app
$ vibe init

# Add missing features
$ vibe add docker
$ vibe add ci
$ vibe add testing

# Build with context
$ vibe run "add caching"
